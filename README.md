# micro-hth-notes

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库中的页面内容彼此独立，不针对特定域名或具体网站，主要用于记录和展示各类笔记、工具页面或轻量级文档。

## 项目简介

`micro-hth-notes` 是一个以 HTML 页面为主要载体的笔记归档仓库。每个页面均可独立访问，适合存放一些无需依赖后端服务的静态内容。仓库本身不提供任何形式的推广或营销功能，仅作为内容存放与发布的载体。

## 目录说明

```
.
├── README.md          # 本文件
├── index.html         （如有）首页或导航页面
├── page1.html         （示例）独立页面 1
├── page2.html         （示例）独立页面 2
└── assets/            （如有）公共资源文件夹
    └── style.css      （示例）样式文件
```

- 每个 HTML 文件代表一个独立的归档页面。
- `assets/` 目录可用于存放多个页面共用的样式、脚本或图片资源，避免重复。
- 建议在 `index.html` 中提供页面列表或导航，方便浏览。

## 页面归档说明

- 所有页面均为纯静态 HTML，不依赖外部数据库或服务器端逻辑。
- 页面内容由仓库维护者自行决定并更新，内容范围包括但不限于技术笔记、工具页面、个人备忘等。
- 页面之间无强关联，可以独立修改或新增，无需同步更新其他文件。
- 归档内容不针对任何特定网站，也不包含任何形式的营销或诱导点击信息。

## 维护说明

- 仓库采用常规 Git 工作流进行维护。
- 新增页面时，请直接在根目录或子目录下创建 `.html` 文件，并适当更新 `index.html` 中的导航（如有）。
- 修改已有页面时，请保持页面标题、编码等基础信息完整。
- 不建议在页面中嵌入外部不可控资源（如外链脚本），以确保页面长期可访问。
- 欢迎通过 Pull Request 提交改进建议（如样式优化、导航完善等），但请保持内容风格一致。
